# User Research Plan

**Purpose:** Systematically gather user insights to inform product decisions, validate features, and improve user experience.

---

## Research Goals

### Primary Goals
1. Understand core user workflows and pain points
2. Validate feature priorities and roadmap decisions
3. Measure user satisfaction and identify improvement areas
4. Discover unmet needs and opportunities

### Research Questions
- What are the most common transaction management workflows?
- Where do users experience friction in their current process?
- Which features provide the most value?
- What would make users recommend GravMap to others?
- Why do users churn (or consider churning)?

---

## User Interview Scripts

### Interview 1: Onboarding & First Impressions (New Users)

**Target:** Users who signed up in the last 7 days
**Duration:** 30 minutes
**Format:** Video call (Zoom/Google Meet)

#### Introduction (2 min)
```
Thanks for taking the time to chat with me today! I'm [Name] from 
the GravMap team. We're doing research to better understand how 
people use our product and where we can improve.

This will take about 30 minutes. There are no wrong answers - we 
just want to hear your honest thoughts. Is it okay if I record 
this for our team to review later?

[Start recording if consent given]
```

#### Background Questions (5 min)
```
1. Tell me about your role in real estate. What do you do day-to-day?

2. How many transactions do you typically handle in a month?

3. What tools or software do you currently use to manage transactions?

4. What's the biggest challenge you face in managing transactions?

5. How did you first hear about GravMap? What made you decide to try it?
```

#### Onboarding Experience (10 min)
```
6. Walk me through your experience signing up. What was easy? 
   What was confusing?

7. Did you complete the onboarding flow? [If no] What stopped you?

8. Did you create your first transaction? [If no] What's preventing you?

9. Did you upload a document for AI extraction? How did that go?

10. Was there anything you expected to see but didn't?
```

#### Value Understanding (5 min)
```
11. In your own words, what do you think GravMap does?

12. What's the most valuable feature you've seen so far?

13. Is there anything that's unclear or confusing about the product?
```

#### Wrap-Up (5 min)
```
14. On a scale of 1-10, how likely are you to continue using GravMap?

15. What's the one thing we could do to make GravMap better for you?

16. Is there anything else you'd like to share that we haven't covered?

Thanks so much for your time! We'll use this feedback to improve 
the product. If you have any questions later, feel free to reach out.
```

### Interview 2: Power User Deep-Dive

**Target:** Users who have created 5+ transactions
**Duration:** 45 minutes
**Format:** Video call with screen sharing

#### Introduction (2 min)
```
Thanks for chatting with us! You've been using GravMap for a while, 
so we really value your insights. I'd love to see how you use the 
product in your daily work.

Is it okay if I record this and share my screen?
```

#### Usage Patterns (10 min)
```
1. Can you show me how you typically use GravMap? Walk me through 
   a recent transaction you managed.

2. How often do you log in? What prompts you to check GravMap?

3. Which features do you use most frequently?

4. Are there features you don't use? Why not?
```

#### Workflow Context (10 min)
```
5. Walk me through your process for a new transaction from start to 
   finish. Where does GravMap fit in?

6. What other tools do you use alongside GravMap? How do they work 
   together?

7. How do you currently handle deadline reminders before using GravMap?

8. What's your typical day like? When do you have time to check on 
   transactions?
```

#### Pain Points & Gaps (10 min)
```
9. What's the most frustrating part of using GravMap?

10. Are there tasks you still do outside of GravMap that you wish 
    you could do inside?

11. Have you ever encountered a situation where GravMap didn't work 
    the way you expected?

12. If you could change one thing about GravMap, what would it be?
```

#### Value & NPS (5 min)
```
13. What's the main benefit you get from using GravMap?

14. On a scale of 0-10, how likely are you to recommend GravMap to 
    a colleague? [Follow up: Why that score?]

15. Have you recommended GravMap to anyone? What did you tell them?
```

#### Future Features (5 min)
```
16. We're considering adding these features [show list]. Which 
    would be most valuable to you? Why?

17. Are there features you'd expect to see that we don't have?

18. What would make you upgrade to a higher tier / stay subscribed 
    for longer?
```

#### Wrap-Up (3 min)
```
19. Is there anything else you'd like to share?

20. Would you be interested in being part of our customer advisory 
    board or beta testing new features?

Thank you so much! Your feedback is incredibly valuable.
```

### Interview 3: Churn/Exit Interview

**Target:** Users who cancelled or haven't logged in for 30+ days
**Duration:** 20 minutes
**Format:** Phone or video call

#### Introduction (2 min)
```
Hi [Name], thanks for talking with me. I noticed you recently 
[cancelled/stopped using] GravMap. We're trying to understand 
what we could have done better. This isn't a sales call - I 
genuinely want to understand your experience.
```

#### Understanding the Decision (10 min)
```
1. What initially attracted you to GravMap?

2. What was your experience like when you first started using it?

3. Walk me through what led to your decision to [cancel/stop using].

4. Was there a specific moment or issue that was the "last straw"?

5. What are you using now instead of GravMap? How is that working out?
```

#### Product Feedback (5 min)
```
6. What did you like about GravMap?

7. What was missing or not working well?

8. Was there anything confusing about the product?

9. How was the pricing relative to the value you got?
```

#### Recovery Opportunity (3 min)
```
10. Is there anything we could do to bring you back?

11. What would GravMap need to have for you to consider it again?

12. Would you recommend GravMap to anyone else? Why or why not?
```

## Usability Testing Sessions

**Duration:** 45-60 minutes
**Tools:** Zoom + screen sharing, or Lookback.io
**Participants:** 5 users per feature tested

### Test Script Template

```

Researcher: [NAME]

### Pre-Test Questions
1. How long have you used GravMap?
2. How often do you [do task related to feature]?

### Tasks
[Give tasks one at a time. Don't help unless stuck for 2+ minutes.]

Task 1: [Description]
- Starting point: [Where to begin]
- Success criteria: [What counts as completion]
- Time taken: ___
- Difficulty rating (1-5): ___
- Notes:

Task 2: ...

Task 3: ...

### Post-Test Questions
1. What was easy about [feature]?
2. What was difficult or confusing?
3. Did anything behave differently than you expected?
4. How would you improve this feature?
5. On a scale of 1-5, how useful is this feature to you?

### Observations
- [Behavior observations]
- [Quotes]
- [Pain points]
- [Errors made]
```

## Feedback Analysis Framework

### Feedback Sources
- In-app feedback widget
- Support tickets
- User interviews
- Survey responses
- NPS comments
- App store reviews
- Social media mentions
- Sales calls

### Categorization System

```
Category
├── Bug/Issue
│   ├── Critical
│   ├── Major
│   └── Minor
├── Feature Request
│   ├── New Feature
│   ├── Enhancement
│   └── Integration
├── UX/UI
│   ├── Confusion
│   ├── Friction
│   └── Suggestion
├── Pricing/Billing
│   ├── Too expensive
│   ├── Billing issue
│   └── Plan change
└── Positive
    ├── Love it
    ├── Specific praise
    └── Testimonial candidate
```

### Processing Workflow

1. **Collect** → All feedback goes to central inbox
2. **Triage** → Categorize and prioritize (daily)
3. **Tag** → Add relevant tags (feature area, user segment)
4. **Analyze** → Weekly review for patterns
5. **Action** → Create tickets / update roadmap
6. **Close Loop** → Respond to user when appropriate
